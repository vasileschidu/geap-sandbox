Core Concepts

Colors

Using and customizing the color palette in Moldova Design System projects.

These primitives define the full visual temperature of the system, from brand expression and interface neutrals to state semantics and overlays. The layout below is designed for fast comparison, so teams can scan hue families horizontally and judge tonal consistency at a glance.

Our brand ramps follow the same progression principle across the system: lower steps are soft surface tints, middle steps carry interaction weight, and higher steps become the dense anchors used for emphasis, contrast, and inverse moments. The example below shows how Blue Sky tokens are applied in real component code.

Blue Sky brand ramp example

/* Component usage with design-system tokens */<button class="brand-button">Continue</button><a class="brand-link">View details</a><p class="text-blue-sky-600">Approved application</p>.brand-button {  background: var(--blue-sky-600);  color: var(--white);  border: 1px solid var(--blue-sky-600);}.brand-button:hover {  background: var(--blue-sky-700);}.brand-button:focus-visible {  box-shadow: 0 0 0 4px var(--blue-sky-150);}.brand-link {  color: var(--blue-sky-600);}/* Utility class generated by the system */.text-blue-sky-600 {  color: var(--blue-sky-600) !important;}

Use the lighter end for focus rings and soft surfaces, keep `600` and `700` for primary interactive states, and reserve the deepest values for strong contrast or brand-heavy surfaces. The generated comparison shades on the larger atlas are preview aids, not first-class shipped tokens.
